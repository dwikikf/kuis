  ✓ No gaps in numbering sequence

  ✓ FIFO lock queue order maintained
  ✓ All student information preserved
  ✓ Error handling without data loss

How LockService Prevents Race Condition:

WITHOUT Lock (PROBLEM):
  Student 1: Read LastRow=1 → Set No=2
  Student 2: Read LastRow=1 → Set No=2 ← DUPLICATE!
  Student 3: Read LastRow=1 → Set No=2 ← DUPLICATE!
  
WITH Lock (SOLUTION):
  Student 1: Lock Acquired → Read=1 → Set No=2 → Release
  Student 2: Lock Acquired → Read=2 → Set No=3 → Release
  Student 3: Lock Acquired → Read=3 → Set No=4 → Release
  
Result: Sequential 2,3,4 ✓ (Not 2,2,2)
